Backend — Low-Level Architectuur

Middleware pipeline, modules, services, events
Middleware Pipeline (elke request)
1
RateLimitMiddleware
Redis sliding window — 60 req/min per IP. Fallback: in-memory buckets.
2
RequestIDMiddleware
Genereert UUID, bindt aan structlog context, retourneert X-Request-ID header.
3
CORS Middleware
Whitelist-based origins. Credentials allowed voor JWT refresh.
4
Slug-in-URL Tenant Resolution
Parent router /schools/{slug} → resolve_tenant_from_path() dependency → request.state.tenant_id. In-memory cache met TTL.
5
Router Dispatch
FastAPI dependency injection: get_current_user(), require_permission(), get_tenant_db()

Toekomst — Microservices Roadmap

Wat nodig is om de monolith op te splitsen (als het moment komt)
Nu niet nodig. Een modulaire monolith is voor 95% van de SaaS-bedrijven de juiste keuze tot tienduizenden gebruikers. De architectuur is voorbereid voor selectief splitsen wanneer het nodig is.
Stappen om te splitsen (geschatte inspanning: 10-15 dagen)
1
Event Bus → Message Queue 2-3 dagen
Vervang in-process event_bus.py door RabbitMQ of Redis Streams. Zelfde interface, andere transport.
2
Email templates naar core/ 1 dag
Verplaats templates uit tenant/notification/ naar core/ om cross-module import te verbreken.
3
Service-to-service → REST/gRPC 2-3 dagen
Vervang directe service calls (attendance → student) door HTTP clients met retry logic.
4
Per-service deployment 2-3 dagen
Aparte Dockerfiles, eigen database connections, CI/CD per service.
5
Integratie testen 2 dagen
End-to-end verificatie van alle inter-service communicatie.
